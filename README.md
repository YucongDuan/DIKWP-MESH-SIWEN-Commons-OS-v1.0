# DIKWP-MESH² SIWEN Commons OS v1.0

**A no-definition, policy-to-protocol, human-control, interoperability and inclusion commons for China's AI+ agent era.**

`SIWEN` is only a project handle referring to the four questions of the intelligent era. It is not a new ontology and does not sit above DIKWP-MESH².

The system compiles policy and scenario signals into observer-, context- and purpose-indexed semantic resources, routes them through all 25 co-equal `DIKWP×DIKWP` transformations, maps public-agent controls to China's seven-part agent-interconnection standard family, and produces deterministic control decisions, traceable receipts, incident bundles, and public-product packages.

## Core outputs

- policy semantic bundles without final concept definitions;
- distributed `PurposeField` and `DecisionBoundary`;
- `AgentPassport`, `CapabilityManifest`, and `ToolLease`;
- five-level gates: `ALLOW / REVIEW / HOLD / BLOCK / KILL`;
- mapping to the seven-part intelligent-agent interconnection standards;
- culture, inclusion, accessibility, energy and public-good extensions;
- 14 deterministic shadow scenarios;
- offline dashboard, JSON Schema, tests and append-only ledger.

## Quick start

```bash
python -m venv .venv
source .venv/bin/activate
pip install -e .[dev]
dikwp-siwen demo --outdir outputs/demo
pytest -q
dikwp-siwen validate --root .
```

Open `dashboard/index.html` for the offline dashboard.
